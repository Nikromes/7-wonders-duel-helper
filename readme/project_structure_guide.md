# Структура проекта

## Файлы

| Файл | Описание |
|------|----------|
| `index.html` | Главная страница — вкладки, модалки, кнопки |
| `styles.css` | Все стили: карточки, модалки, вкладки, DLC бейджи |
| `script.js` | Логика: отрисовка карт, вкладки, поиск, калькулятор эпох |
| `data.js` | Все данные игры: жетоны, чудеса, гильдии, колоды эпох |
| `scan.js` | Фото-сканирование стола через AI + модалка настроек |
| `scan-prompt.js` | AI промпт-шаблон для распознавания фото |
| `ai-config.js` | Настройка модели Gemini (модель, endpoint) |
| `api-key.js` | Управление API-ключом (localStorage + фолбэк) |
| `.gitignore` | Файлы исключённые из Git |

## Порядок загрузки скриптов в index.html

```
api-key.js → ai-config.js → scan-prompt.js → data.js → script.js → scan.js
```

Порядок важен: каждый скрипт зависит от предыдущих (api-key → ai-config → scan).

## Структура данных в data.js

### Жетоны и Чудеса
```javascript
{ title: "Название", desc: "Описание эффекта", tags: [...], color: "token", cost: [...], dlc: "Pantheon" }
```

### Карты эпох (predictorDeck)
```javascript
{ id: "a1_br1", title: "Название", type: "Эффект", color: "brown", cost: [...], chainGiv: "Символ", chainReq: "Условие" }
```

### DLC поддержка
Поле `dlc: "НазваниеDLC"` — элемент из дополнения. CSS-класс `.dlc-названиеdlc` задаёт цвет бейджа.

## Гильдии
Данные гильдий хранятся **один раз** в `gameData.guilds` и автоматически подключаются в калькулятор Эпохи III через `script.js`.
